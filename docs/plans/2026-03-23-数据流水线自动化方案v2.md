## 一、概述

本文档描述 A股数据更新工作流的自动化方案，解决三个串型数据更新任务的调度 + 小金监控问题。

### 1.1 工作流结构

```
┌─────────────────────────────────────────────────────────────────┐
│                      数据流水线 (Dailypipeline)                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Step 1          Step 2            Step 3           Step 4      │
│  ┌─────┐         ┌─────┐          ┌─────┐         ┌─────┐      │
│  │stock│         │daily│          │signals│        │trade│      │
│  │_info│   ──▶   │price│    ──▶  │      │   ──▶  │     │      │
│  │更新 │         │下载 │          │计算  │         │报告 │      │
│  └─────┘         └─────┘          └─────┘         └─────┘      │
│                                                                  │
│  依赖:无         依赖:Step1        依赖:Step2     依赖:Step3   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 四个步骤

| 步骤 | 脚本 | 输入依赖 | 输出 |
|------|------|---------|------|
| **Step 1** | `fetcher_all_stockV3.py` | 无 | stock_info 表 |
| **Step 2** | `fetcher_daily_priceV4.py` | stock_info 完成 | daily_price 表 |
| **Step 3** | `signals/scan_signals_v2.py` | daily_price 完成 | daily_signals 表 |
| **Step 4** | `stock_trade skill` | signals 完成 | 交易执行 + 报告 |

## 二、数据库设计

### 2.1 任务状态记录表 `data_pipeline_run`

```sql
CREATE TABLE data_pipeline_run (
    id              INTEGER PRIMARY KEY,
    pipeline_id     VARCHAR,          -- 流水线ID (如 "daily_20260323")
    pipeline_name   VARCHAR,           -- 流水线名称 (如 "daily", "weekly")
    step_name       VARCHAR,           -- 步骤名: "stock_info", "daily_price", "signals", "trade"
    step_order      INT,              -- 步骤序号: 1, 2, 3, 4
    
    -- 时间戳
    created_at      TIMESTAMP,
    started_at      TIMESTAMP,
    completed_at    TIMESTAMP,
    duration_sec    FLOAT,
    
    -- 运行参数
    params          JSON,             -- {"start_date": "20260323", "end_date": "..."}
    
    -- 状态
    status          VARCHAR,          -- pending, running, success, failed, skipped
    
    -- 结果
    records_count   INT,              -- 处理的记录数
    error_message   TEXT,
    
    -- 前置条件检查
    depends_on      VARCHAR,          -- 依赖的step_name
    dependency_met  BOOLEAN,          -- 依赖是否满足
    
    UNIQUE(pipeline_id, step_name)
);
```

### 2.2 泛化步骤日志表 `step_update_log`（V2 新增）

```sql
CREATE TABLE step_update_log (

    update_type         VARCHAR,       -- 'full', 'incremental', 'daily'

    update_time         TIMESTAMP,     -- 检测时间
    start_time          TIMESTAMP,     -- 更新开始时间

    duration_sec        FLOAT,         -- 更新耗时（秒）
    
    -- 数量统计
    expected_count      INT,           -- 预期数量
    actual_count        INT,           -- 实际入库数量
    
    -- 状态
    is_success          BOOLEAN,       -- 是否成功

    -- 步骤特定字段
```

## 三、小金监控接口（V2 新增）

### 3.1 设计原则

小金通过调用 `workflow_scheduler.py --xiaoji-check` 获取流水线状态，返回 JSON 格式。

### 3.2 CLI 接口

```bash
$ python scripts/workflow_scheduler.py --xiaoji-check [--date YYYYMMDD]

# 返回示例（成功）
{
  "pipeline_id": "daily_20260323",
  "date": "2026-03-23",
  "status": "success",
  "last_step": "signals",
  "steps": {
    "stock_info": {"status": "success", "records": 5000, "duration": 45.2},
    "daily_price": {"status": "success", "records": 15000, "duration": 1850.3},
    "signals": {"status": "success", "records": 4500, "duration": 320.5}

# 返回示例（失败）
{
  "pipeline_id": "daily_20260323",
  "date": "2026-03-23",
  "status": "failed",
  "failed_step": "signals",
  "error": "计算超时，股票 600001 指标异常",

  "date": "2026-03-23",

}
```

### 3.3 实现代码

```python
# workflow_scheduler.py 新增方法

def xiaoji_check(date: str = None) -> dict:
    """小金监控接口 - 返回流水线状态"""
    import json
    from datetime import datetime
    
    today = date or datetime.now().strftime('%Y%m%d')
    pipeline_id = f"daily_{today}"
    
    result = {
        "pipeline_id": None,
        "date": today,
        "status": "no_data",
        "notify_required": False,
        "xiaoji_check_time": datetime.now().isoformat()
    }
    
    # 查询今日流水线状态
    steps = get_pipeline_steps(pipeline_id)
    if not steps:
        # 无今日数据，查询最后成功日期
        last_success = get_last_success_date()
        result["last_success_date"] = last_success
        result["notify_required"] = True
        result["message"] = f"{(datetime.now().date() - last_success).days}天未更新数据，请检查"
        return result
    
    # 构建步骤状态
    step_status = {}
    last_step = None
    has_failed = False
    
    for step in sorted(steps, key=lambda x: x['step_order']):
        step_name = step['step_name']
        step_status[step_name] = {
            "status": step['status'],
            "records": step.get('records_count'),
            "duration": step.get('duration_sec'),
            "error": step.get('error_message') if step['status'] == 'failed' else None
        }
        if step['status'] == 'success':
            last_step = step_name
        if step['status'] == 'failed':
            has_failed = True
            result["failed_step"] = step_name
            result["error"] = step['error_message']
    
    result["pipeline_id"] = pipeline_id
    result["steps"] = step_status
    result["last_step"] = last_step
    
    if has_failed:
        result["status"] = "failed"
        result["last_success_step"] = last_step
        result["notify_required"] = True
    else:
        # 检查是否所有步骤都完成
        all_done = all(s['status'] in ('success', 'skipped') for s in steps)
        result["status"] = "success" if all_done else "running"
    
    return result
```

## 四、告警通知机制（V2 新增）

### 4.1 小金主动检查 + 通知

**触发时机**：
1. **每日定时检查**（cron 18:30）：小金调用 `--xiaoji-check`（数据更新从 18:00 开始）
2. **发现问题时**：立即通知用户

**通知条件**：
| 条件 | 是否通知 |
|------|---------|
| 今日流水线不存在 | ✅ 通知（"3天未更新数据"）|
| 今日流水线失败 | ✅ 通知（"signals 步骤失败，错误原因"）|
| 今日流水线成功 | ✅ 通知（"所有步骤成功，今日流水线完成"）|
| 昨日数据缺失 | ✅ 通知（"昨日数据未更新"）|

### 4.2 通知消息格式

**失败通知**：
```
🚨 数据流水线异常 | 2026-03-23

❌ 失败步骤: signals
📝 错误原因: 计算超时，股票 600001 指标异常
⏰ 上次成功: daily_price (2026-03-23 16:35)

📊 流水线状态:
  ✅ stock_info - 成功 (5000条, 45秒)
  ✅ daily_price - 成功 (15000条, 1850秒)
  ❌ signals - 失败

⚡ 需要人工介入处理
```

**成功通知**：
```
✅ 数据流水线完成 | 2026-03-23

📊 今日数据更新成功！

流水线状态:
  ✅ stock_info - 成功 (5000条, 45秒)
  ✅ daily_price - 成功 (15000条, 1850秒)
  ✅ signals - 成功 (4500条, 320秒)
  ✅ trade - 完成 (买入1只, 卖出0只)

⏱ 总耗时: 37分钟
💰 账户状态: 仓位78%, 总盈亏-2.3%

📈 数据已就绪，明日交易可执行
```

**数据缺失通知**：
```
⚠️ 数据更新预警 | 2026-03-23

📊 数据已 3 天未更新
最后成功日期: 2026-03-20

🔍 可能原因:
  - 定时任务未执行
  - Mac 进入休眠
  - 网络/数据源异常

⚡ 请检查后台任务状态
```

```python
def check_pipeline_status():
    """检查今日数据流水线状态"""
    result = subprocess.run([
        sys.executable, PIPELINE_SCRIPT,
        "--xiaoji-check",
        "--date", datetime.now().strftime('%Y%m%d')
    ], capture_output=True, text=True, timeout=30)
    
    import json
    status = json.loads(result.stdout)
    return status

def notify_if_needed(status: dict):
    """根据状态决定是否通知（成功/失败都会通知）"""
    if status['status'] == 'success':
        send_success_notification(status)  # 发送成功通知
    elif status['status'] == 'failed':
        send_failure_notification(status)  # 发送失败通知
    elif status['status'] == 'no_data':
        send_no_data_notification(status)  # 发送预警通知

def send_success_notification(status: dict):
    """发送成功通知"""
    steps = status.get('steps', {})
    total_duration = sum(s.get('duration', 0) for s in steps.values())
    
    message = f"""✅ 数据流水线完成 | {status['date']}

📊 今日数据更新成功！

流水线状态:
"""
    for step_name, step_data in steps.items():
        emoji = "✅" if step_data['status'] == 'success' else "❌"
        records = step_data.get('records', '-')
        duration = step_data.get('duration', 0)
        message += f"  {emoji} {step_name} - {step_data['status']} ({records}条, {duration:.0f}秒)\n"
    
    message += f"""
⏱ 总耗时: {total_duration:.0f}秒
💰 数据已就绪，明日交易可执行
"""
    send_feishu_message(message)
```
