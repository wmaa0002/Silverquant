## 2. 数据层 (Data Layer)

### 2.1 外部数据源

| 数据源 | 类型 | 限制 | 代码格式 |
|--------|------|------|----------|
| **tushare** | Pro API | 50次/分钟 | `600000.SH`, `000001.SZ` |
| **baostock** | 免费 | 无明确限制 | `sh.600000`, `sz.000001` |
| **akshare** | 免费 | 需代理 | `000001.SZ`, `600000.SH` |

**配置** (`config/settings.py`):
```python
DATA_SOURCE_PRIORITY = ['tushare', 'baostock']  # 数据源优先级
FAILOVER_ERROR_RATE_THRESHOLD = 0.30  # 30%错误率触发故障转移
FAILOVER_RESPONSE_TIME_THRESHOLD = 10.0  # 10秒响应触发故障转移
```

### 2.2 数据适配器 (Adapters)

| `financial.py` | `TushareIncomeFetcher` | 利润表 | `fetch_by_stock()`, `fetch_all()` |
| `financial.py` | `TushareBalanceSheetFetcher` | 资产负债表 | `fetch_by_stock()` |
| `financial.py` | `TushareCashFlowFetcher` | 现金流量表 | `fetch_by_stock()` |

| 文件 | 类/函数 | 职责 | 关键方法 |

| 字段 | Tushare单位 | Database单位 | 转换 |
|------|-------------|--------------|------|
| `vol` | 手 (1手=100股) | 股 | ×100 |

### 2.3 数据更新器 (Updaters)

#### 统一更新器 DWDFetcher (`data/updaters/fetcher_dwd.py`)

**核心类:**
```python
class DWDFetcher:
    # 日线数据
    def update_daily(start_date, end_date)           # 按日期顺序更新
    def update_daily_parallel(start, end, workers)    # 多进程并行
    def update_daily_by_stock(start, end, workers)    # Baostock按股票更新
    
    # 市场指标
    def update_daily_basic(start_date, end_date)     # dwd_daily_basic
    
    # 复权因子
    def update_adj_factor(start_date, end_date)      # dwd_adj_factor
    
    # 财务数据
    def update_income(ts_code)                        # 利润表
    def update_balancesheet(ts_code)                  # 资产负债表
    def update_cashflow(ts_code)                      # 现金流量表
    def update_financial_multiprocess(workers)        # 多进程全量更新
    
    # 指数数据
    def update_index(index_code, start, end)          # dwd_index_daily
    
    # 基础数据
    def update_stock_info()                           # dwd_stock_info
    def update_trade_calendar(start, end)              # dwd_trade_calendar
    
    # 增量更新
    def update_incremental(data_type)                  # 自动检测最新日期后更新
```

**使用方式:**
```bash
# 增量更新 (自动检测)
python fetcher_dwd.py --incremental --data-type daily

# 全量更新
python fetcher_dwd.py --full --data-type all --start-date 20240101

# 指定数据类型
python fetcher_dwd.py --data-type daily --start-date 20260330 --end-date 20260406
python fetcher_dwd.py --data-type stock_info
python fetcher_dwd.py --data-type trade_calendar --start-date 20200101
```

```
    ├── B2_strategy_module.py        # B2买入策略
```

### 3.2 技术指标 (basic_module.py)

**输出字段:**

| 类别 | 指标 |
|------|------|
| **价格/OHLC** | `open, high, low, close, volume, prev_close, 涨幅, 振幅, 波幅, 波动率` |
| **均线** | `ma3, ma5, ma6, ma10, ma12, ma14, ma20, ma24, ma28, ma30, ma50, ma57, ma60, ma114` |
| **自定义线** | `知行短期趋势线` (EMA10的EMA10), `知行多空线` ((MA14+MA28+MA57+MA114)/4) |
| **MACD** | `dif` (EMA12-EMA26), `dif_arr`, `dea` |
| **KDJ** | `k, d, j` |
| **RSI** | `rsi1`(3日), `rsi2`(14日), `rsi3`(28日), `rsi4`(57日) |
| **量能均线** | `vol_ma5, vol_ma10, vol_ma20, vol_ma60` |
| **BBI** | `bbi`, `前20日BBI` |
| **特殊K线** | `大长阳, 大长阴, 关键K, 暴力K, 参考成交量` |

### 3.3 策略信号

#### B1 买入策略

**触发条件:**
- KDJ_J < 13
- MACD多头排列 (dif >= 0)
- 知行短期趋势线 > 知行多空线
- B1总分 >= 8

**评分规则 (39项子条件):**

| 条件 | 分数 |
|------|------|
| DIF >= 0 | +0.6 |
| C > MA60 | +1.5 |
| 涨幅 -2%~1.8% | +1.5~-3 |
| 振幅<7 / <4 | +0.5/+0.8 |
| RSI1<20 / <23 | +0.8/+0.7 |
| 地量判断 (10窗口) | +0.3/项 |
| 倍量柱+阳线 | +0.4~1 |
| DIF底背离 | -0.5~-1 |
| 放量上涨 | +0.5~1.4 |
| 跳空阳线 | -1.2~-2.2 |

#### B2 买入策略

**触发条件:**
- 前日J <= 21 或 前日RSI1 <= 21
- 今日放量 (VOL > REF(VOL,1))
- 涨幅 > 3.95%
- VOL > VOL_MA60
- C > O

**评分规则 (31项子条件):**

| 条件 | 分数 |
|------|------|
| J>80 | -3 |
| DIF>=0 | +0.3 |
| MA10/20/50多头 | +0.6 |
| C > 知行多空线 | +0.3 |
| RSI1>枢轴线 | +0.5~+0.8 |
| 上影线<=1/4 | +1.5 |
| VOL>VOL_MA60/1.25x/1.5x | +0.4~+1.8 |
| C=HHV(90/60/30) | +0.7/项 |
| 暴力K | +1 |
| 大长阴 | -0.4~-2 |

#### BLK 暴力K策略

**触发条件:**
1. C > REF(C,1) (今日上涨)
2. VOL > 参考成交量 × 1.8 (放量1.8倍)
3. 涨跌幅 > 4%
4. 上影线 <= K线长度 / 3.5
5. 前9日涨幅 < 4
6. 前4日涨幅 < 4
7. VOL > VOL_MA60

**分数:** 满足条件则固定 7 分

#### BLKB2 组合策略

**触发条件:**
- MACD多头 (dif >= 0)
- 知行短期趋势线 > 知行多空线
- B2总分 >= 8
- 暴力K = True
- 倍量柱 = True (VOL > REF(VOL,1)×1.8 AND C>O AND C>REF(C,1))
- J拐头向上 = True (今日J > 昨日J AND 昨日J < 前日J)

**评分公式:**
```
score_blkB2 = score_blk×0.5 + score_b2×0.6 + score_blz×0.2 + score_jt×0.1
```

#### SCB 沙尘暴策略

**触发条件:**
1. 地量等级 >= 30
2. 非ST股
3. 前60日非阴
4. COUNT(关键K OR 暴力K, 60) >= 1
5. 过去5日满足地量基础条件 AND 暴力K AND 知行多空线 > REF(知行多空线,60)

**评分:**

| 条件组合 | 分数 |
|----------|------|
| 地量+暴力K+多头 | 10 |
| 地量+暴力K | 7 |
| 地量+多头 | 4 |
| 暴力K+多头 | 5 |
| 地量 only | 2 |
| 暴力K only | 3 |

#### DZ30 单针30策略

**触发条件:**
1. 长期KD >= 80 (21日KDJ)
2. 短期KD <= 30 (3日KDJ)
3. 价格在趋势线上 (C > 知行短期趋势线)
4. 趋势多头 (知行短期趋势线 > 知行多空线)
5. 有倍量柱 (近20日倍量柱>=1)
6. 前20日非阴

**分数:** 满足条件则固定 5 分

#### S1 卖出策略

**触发条件:**

| S1分数 | 操作 |
|--------|------|
| >= 10 | 全卖 (Full sell) |
| >= 5 and < 10 | 卖半 (Half sell) |
| 跌破多空线 (连续2日) | 清仓 |

**评分规则:**

| 条件 | 分数 |
|------|------|
| C<O AND HIGH=HHV(H,60) AND (前10日涨>10% OR 前50日涨>50%) | 6.5~10 |
| HHV(H,4)=HHV(H,60) AND HIGH!=HHV(H,60) AND VOL>MA5/MA10 AND 涨幅<-3% AND C<O AND VOL>=REF(VOL,3)×1.2 | 12 |
| 条件1基础上加分 | +1.5 |
| J<K<D | +0.8 |

**附加卖出信号:**
- `止损`: 盘中最低价 < 买入价 × 0.97 (3%止损)
- `跌破多空线`: 观察期机制 (第一天下跌=观察, 连续第二天=卖出)

### 4.3 策略实现示例 (天宫B1策略v2.1.py)

```python
class 天宫B1策略v2(bt.Strategy):
    def __init__(self):
        self.inds = {}
        for d in self.datas:
            self.inds[d] = {}
            # 初始化指标
            self.inds[d]['ma60'] = bt.indicators.SMA(d.close, period=60)
            self.inds[d]['macd'] = bt.indicators.MACD(d.close)
            self.inds[d]['kdj'] = bt.indicators.StochasticSlow(d.close)
            # ... 更多指标
    
    def next(self):
        for d in self.datas:
            # 获取指标值
            j = self.inds[d]['kdj'].j
            dif = self.inds[d]['macd'].dif
            
            # B1买入条件
            if j < 13 and dif >= 0 and self.inds[d]['短期趋势线'] > self.inds[d]['多空线']:
                score_b1 = self.calculate_b1_score()
                if score_b1 >= 8:

            # S1卖出条件
            if self.get_position(d) > 0:
                score_s1 = self.calculate_s1_score()
                if score_s1 >= 10:
                    self.close(d)
                elif score_s1 >= 5:
                    self.close(d, size=self.position.size/2)
```

### 5.2 回测引擎 (engine.py)

**核心类:** `BacktestEngine`

```python
class BacktestEngine:
    def __init__(
        self,
        initial_cash: float = 100000.0,   # 初始资金
        commission: float = 0.0003,        # 手续费 0.03%
        stamp_duty: float = 0.001,          # 印花税 0.1%
        slip_page: float = 0.001            # 滑点 0.1%
    ):
        self.cerebro = bt.Cerebro()
        self.cerebro.broker.setcash(initial_cash)
        self.cerebro.broker.setcommission(commission=commission)
        self.cerebro.broker.set_slippage_perc(slip_page)
        self._add_analyzers()
    
    def add_data(df, name, fromdate, todate):
        """添加数据源"""
        data = bt.feeds.PandasData(dataname=df, ...)
        self.cerebro.adddata(data, name=name)
    
    def add_strategy(strategy_class, **kwargs):
        """添加策略"""
        self.cerebro.addstrategy(strategy_class, **kwargs)
    
    def run():
        """执行回测"""
        results = self.cerebro.run()
        return self._extract_results(results[0])
```

**内置分析器:**
- `Returns` - 收益率分析
- `SharpeRatio` - 夏普比率
- `DrawDown` - 最大回撤
- `TradeAnalyzer` - 交易统计
- `TimeReturn` - 日收益
- `PositionsValue` - 持仓价值

### 5.3 回测数据流

```
┌─────────────────────────────────────────────────────────────────────┐
│                      回测执行流程                                     │
└─────────────────────────────────────────────────────────────────────┘

1. 数据加载

         │
         ▼

         │
         ▼

         │
         ▼

         │
         ▼

         │
         ▼
   cerebro.addstrategy() → 注册策略到backtrader

3. 回测执行
   engine.run() → cerebro.run()
         │
         ▼
   For each bar (next()):
     ├── time_filter() → 跳过节假日、14:30前
     ├── 无持仓: buy_condition() → score>=threshold → buy
     └── 有持仓: sell_condition() → True → sell

     - notify_order() → 计算盈亏, 记录

     - final_value = cerebro.broker.getvalue()

     - strat.trade_records → result['trades']

6. 持久化
```

### 6.2 主交易流程 (run_stock_trade.py)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    run_stock_trade.py 交易流程                      │
└─────────────────────────────────────────────────────────────────────┘

1. 自动检测最新交易日
   latest_signal_date ← MAX(date) FROM daily_signals
   latest_price_date ← MAX(trade_date) FROM dwd_daily_price
   TRADE_DATE ← min(latest_signal_date, latest_price_date)

2. 获取持仓
   SELECT code, name, strategy, shares, buy_price 
   FROM positions WHERE status = 'holding'

3. 卖出信号扫描
   for each position:
       SELECT signal_s1_full, signal_s1_half, signal_跌破多空线, signal_止损,
              signal_sell_b1, signal_sell_b2, ...
       FROM daily_signals WHERE code = ? AND date = TRADE_DATE
       
       通用卖出信号 (所有持仓适用):
       - signal_s1_full=True → 全卖
       - signal_s1_half=True → 卖半
       - signal_跌破多空线 → 观察/卖出
       - signal_止损=True → 止损
       
       策略专属卖出信号:
       - B1持仓 → B1卖出信号
       - B2持仓 → B2卖出信号
       - ...

4. 执行卖出
   UPDATE positions SET status='sold', sell_date=TRADE_DATE, ...
   WHERE code = ?

5. 重新计算仓位
   available_cash ← 当前可用资金
   target_position ← 500000 × 0.85 = 425000
   can_buy ← target_position - 已持仓市值

6. 买入信号扫描
   SELECT code, name, score_b1, score_b2, ...
   FROM daily_signals 
   WHERE date = TRADE_DATE 
     AND signal_buy_* = True
     AND code NOT IN (当前持仓)
   ORDER BY (策略数×10 + 最高分数) DESC

7. 执行买入
   按 PER_STOCK_MIN(15%) ~ PER_STOCK_MAX(30%) 分配资金
   INSERT INTO positions (code, name, strategy, shares, buy_price, ...)

8. 更新持仓快照
   python update_portfolio_daily.py

9. 生成交易报告
   → obsidian/openclaw-skill/量化交易/日交易报告/{TRADE_DATE}.md

10. 交易审计
    python audit_trade.py
```
